# 阅读选区加入 AI 方案归档

## 目的

统一存放已经过时的“阅读选区加入 AI”改造方案，避免后续继续按旧思路开发。

## 归档范围

- `2026-06-double-click-near-selection.md`
- `2026-06-selection-hit-area-spec.md`

## 为什么归档

这两份方案都基于一条已经确认不再采用的思路：

- 在原有“小菜单加入对话”之外，再新造一套双击命中与切换逻辑

这个方向的问题已经确认：

- 改动面过大
- 容易和原有选区状态分叉
- 容易丢失原本按钮链路已有的能力

## 当前采用的方向

后续统一采用“最小改造”原则：

- 不新造第二套选区状态
- 不改阅读器底层选区事件链
- 只复用原本“小菜单加入对话”的现成动作
- 只把触发方式尽量小范围地改到现有 UI 层

## 说明

这些归档文档保留，只用于追溯，不作为当前开发依据。
